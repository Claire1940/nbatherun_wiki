| Step | Target | Recommended 2026 Baseline |

2. Prioritize game traffic over streaming devices.
3. Reboot router before major update windows.
4. Use 5 GHz Wi-Fi only if Ethernet is unavailable.
5. Keep DNS settings consistent (don’t swap frequently unless troubleshooting).

> **💡 Tip:** If your matches feel delayed only at night, test at three time windows (afternoon, evening, late night) and track ping variance. This can reveal ISP congestion patterns faster than random rebooting.

## Troubleshooting Disconnects, Failed Sync, and Queue Errors

Even with strong internet, live-service sports titles can fail at login, matchmaking, or end-of-match reporting. Use this order to isolate the issue quickly.

### Fast triage sequence (5-10 minutes)

1. Check service status channels first (developer social and platform service pages).
2. Restart game client, then relaunch with no background downloads.
3. Power cycle modem/router (60 seconds off).
4. Test a quick public match before ranked/event entries.
5. Switch from Wi-Fi to Ethernet to confirm local interference.
6. Re-authenticate account if progression fails to sync.

### Common issue patterns

| Symptom | Most Likely Cause | First Fix to Try |
|---|---|---|
| **Can log in, cannot queue** | Regional server overload | Change region to nearest alternate |
| **Queue pops then fails** | NAT/port conflict | Reboot router, verify NAT type |
| **Match completes, no rewards** | Delayed server sync | Wait 10 minutes, relog once |
| **Rubber-banding gameplay** | Wi-Fi interference/jitter | Move to Ethernet or 5 GHz clean channel |
| **Frequent mid-match disconnects** | ISP instability | Run packet loss test, contact ISP |

If your goal is consistent progress in **NBA The Run online only**, keep a short log: date, time, mode, ping, and issue type. This makes support tickets much more actionable and speeds up resolution.
